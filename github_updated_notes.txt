=================
	Git Architecture
=================

1)  Working Tree.

2)  Staging Area.

3) Local Repo.

4) Central Repo.


#Working Tree	:- this is your current workspace where project files are created, edited and deleted. this code directly is not eligible to commit.

#Staging area	:- it is the middle layer beween your working tree directory and local repo. it represents which files are eligible to commit.

#Local Repo :-	The place where your commits are permantly stored on your local machine. For each commit one unique commit is will be generated.

#Central Repo :- Hosted on Github.com and used for collaboration among multiple developers.



working tree  ---->  Staging Area --------->  Local repo -----------> Centra Repo


================
	Git Commands
================

# git init : To initalize new git repository.  it creates .git folder in working tree. .git acts like brain of git to co-ordinate with your repo.

	$ git init

#	git status: Display working tree status. (staged/unstaged)

	$	git status

# git add: Add files to staging area. (move files from working tree to staging area)

	$	git add <file-name>

	$ git add .       Will add  all files together in staging area.  (git add .)

# git commit: commit staged files at local repo.

	$ git commit -m "commit message"

# git push: publish local commits to central repo.


# git diff: display modified differences in working tree.

	$ git diff


# git log: To display log history.

 	$	git log -<number>/ $ git log -5 :to get history of only last 5 commits


 #	git log --oneline :- display commit history in single line.

 	$git log --oneline -5: to get the last 5 commit history in single line

 #	git restore --staged <file> : will moe back files from staged area to working tree/unstaged area.

 	$	git restore --staged Third.txt

 # git restore <file>	:	 to discard changes in working directory

 # git branch : list branches, point to current branch

 # git branch -r :  Will show all available branches on central repo (github.com).

 #	git checkout <branch-name> : will switch to given branch.

 #	git pull : Download latest changes from central repo to working tree directly.

## Note : When we use git pull there is a chance of getting merge conflicts.

 =================================
 Q. How to remove git local commits?
 =================================

=> Git reset: Using git reset command we can remove our local commits.

=> Git Reset we can do in 2 ways.

	a) soft reset.

	b) hard reset.

=>soft reset means only local commit will be deleted but file/code changes will be still available in staging area.

	$	git reset --soft HEAD~1

=>hard reset means only local commit will be deleted and  file/code changes also will be deleted from working tree.

	$	git reset --hard HEAD~1


==================================
Q. How to revert git central repo commits?
==================================

=> Using 'git revert' command we can revert code changes from central repo.

	$ git revert <commit-id>

Note:- After git revert execution we need to excute git push command.

How to push entire project to git hub ?
=========================================

Step-1 :: Create new github repository. It will show git commands like below

```
echo "# 302-SB-App" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/ashokitschool/302-SB-App.git
git push -u origin main

```

Note: Instead of adding only 'git add README.md' file to staging area we can add all files by using 'git add .' like below


echo "# Infeanet_2026" >> README.md
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/dnyaneshrt/Infeanet_2026.git
git push -u origin main


Step-2 :: Execute above all commands in project directory then our project files will be pushed to github central repo.

============================
How to merge commits in local repo
============================

=> By using 'Git merge' command we can merge changes from one branch to another branch in local repo.

#create new branch
	$git branch feature/ag123_bug_fix

# create ag123.txt file and commit
	$touch ag123.txt
	$git add ag123.txt
	$git commit -m 'ag123.txt file added to fix bugs'

#switch to main branch

	$git checkout main

#merge feature branch changes into main

	$git merge feature/ag123_bug_fix


============
Git cherry-pick
============

=> When we use 'git merge' command, by deafault all commits will merge from one branch to another branch.

=> If we have requirement to merge specific commit from one branch to another branch then we use 'git cherry-pick'

	$git checkout -b <brach-name> - will create branch and switch
	git checkout -b feature/ronak_module

	$git cherry-pick <commit-id>

	$git cherry-pick <commid-id> <commit-id> ...  <commit-id>

	$git cherry-pick <start-commit-id>^.. <end-commit-id>
